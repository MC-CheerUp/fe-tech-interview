## 쿠키(Cookies)란?

-   웹 서버가 브라우저에 전송하여 저장하는 작은 데이터 파일
-   일반적으로 4KB 이하로 제한된다.
-   만료
    -   세션 쿠키: 브라우저 종료 시 삭제된다.
    -   영속 쿠키: 지정된 만료 날짜까지 유지된다.
-   용도
    -   사용자 인증 (로그인 상태 유지)
    -   사용자 추적 및 분석
    -   사용자 선호 설정 저장
-   특징
    -   HTTP 요청마다 서버로 자동 전송된다.
    -   서버와 클라이언트 모두 접근 가능하다.
    -   도메인 및 경로별로 설정 가능하다.
-   보안
    -   HttpOnly 플래그로 JavsScript 접근을 막을 수 있다.
    -   Secure 플래그로 HTTPS 연결에서만 전송되도록 할 수 있다.
-   단점
    -   크기 제한으로 대량의 데이터 저장에 부적합하다.
    -   매 요청마다 전송되어 대역폭을 차지한다.

## 세션(Session)이란?

-   서버 측에서 유지되는 사용자별 상태 정보이다.
-   작동 방식
    1. 사용자 로그인 시 서버에서 고유한 세션 ID 생성
    2. 세션 ID를 클라이언트에 전송 (주로 쿠키 이용)
    3. 실제 데이터는 서버에 저장
-   용도
    -   사용자 인증 정보 저장
    -   장바구니 기능
    -   임시 데이터 저장
-   장점
    -   보안성이 높다 (중요 정보를 서버에 보관).
    -   쿠키보다 큰 용량의 데이터 저장이 가능하다.
-   단점
    -   서버 리소스를 사용한다.
    -   다중 서버 환경에서 세션 공유 문제가 발생할 수 있다.
-   만료
    -   서버에서 세션 타임아웃을 설정할 수 있다.

## 로컬스토리지

-   HTML5에서 제공하는 클라이언트 측 저장소.
-   일반적으로 5-10MB (브라우저에 따라 다르다)
-   브라우저를 닫아도 데이터가 유지된다.
-   용도
    -   오프라인 데이터 저장
    -   웹 애플리케이션의 상태 저장
    -   캐싱
-   특징
    -   JavaScript를 통해 접근한다.
    -   동일 출처(Same Origin) 정책이 이용된다.
    -   키-값 쌍으로 데이터를 저장한다.
    -   문자열 형태로만 저장 가능하다 (객체는 JSON으로 변환 필요).
-   장점
    -   서버로 자동 전송되지 않아 대역폭을 절약한다.
    -   쿠키보다 큰 용량의 데이터를 저장할 수 있다.
-   단점
    -   JavaScript로 접근 가능하여 XSS 공격에 취약할 수 있다.
    -   동기적으로 작동하여 대량의 데이터 처리 시 성능에 영향을 줄 수 있다.
