# 👔 대화형 면접 스크립트 저장소

<br />

### 🖐🏻 How to Use


1️⃣ 각자의 닉네임으로 branch 를 생성합니다. <br/>

2️⃣ 최상위 폴더(`직무면접`, `문화면접`, `협업면접`, `CS면접`) 중, 작성할 스크립트의 주제의 분류에 맞는 위치로 이동합니다. <br/>

3️⃣ 자신의 닉네임으로 된, 폴더를 생성 후 이동합니다. <br/>
  &nbsp; → `📁 직무면접/Amy`

4️⃣ 스크립트 주제로 파일이름을 작성합니다. <br/>
  &nbsp; → `📁 직무면접/Amy/Attribute와_Property의_차이.md` <br/>

5️⃣ 파일 작성 후, commit 합니다. <br/>

6️⃣ PR 생성 시 반드시 대괄호 안에 본인 닉네임을 대문자로 적어주세요. <br/>
  &nbsp; → `[Amy] Attribute와 Property의 차이` <br/>

<br />
<br />

### 🚦 RULE

branch 명은 소문자로 쓸 것 <br />

MD 파일 제목에서 띄어쓰기는 언더스코어(_)로 처리할 것 <br />

commit 제목은 파일이름과 동일하게 작성할 것 <br />

PR 제목은 반드시 대괄호 안의 본인 닉네임으로 시작할 것 <br />


<br/>


#### 📄 MarkDown

최상위 폴더는 4개 입니다. (직무 면접 / 문화 면접 / 협업 면접 / CS 면접) <br />
각각의 최상위 폴더 안에 본인 닉네임(대문자)의 하위 폴더를 생성해 공부한 내용을 공유해요. <br />

<br/>

#### 👥 PR

마크다운 파일에는 공부한 내용을, PR review에는 질문에 대한 답을 적어주세요. <br />
PR을 올린 사람의 페어는 면접자로써 관련 질문에 대한 답을 달며 질의응답을 이어나갑니다. <br />

<br/>

_(👇 면접관 script 작성 시, 아래를 복사해주세요.)_
```md
<div align="end">
<h3> $\color{#2699E6}{🎤 면접관}$ </h3>

<!-- 면접관 입장에서, 질문을 작성해주세요. -->

</div>

<br/>
```

<br/>

_(👇 면접자 script 작성 시, 아래를 복사해주세요.)_
```md
<div align="start">
<h3>  $\color{#FFA832}{🤚 면접자}$ </h3>

<!-- 면접자 입장에서, 질문을 작성해주세요. -->

</div>
```
